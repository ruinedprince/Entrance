A entrance é um portal de musica eletrônica com diversos recursos, junto de uma plataforma de venda de ingressos.

A missão da entrance é ser a maior plataforma dedicada a musica eletrônica reunindo a maior quantidade de eventos, fotos e conteúdo.
Nos acreditamos que estar junto com o produtor é um grande diferencial auxiliando na criação de conteúdos, marketing e divulgação.
Defendemos a transparência total na administração do evento para que o produtor possa tomar as melhores decisões. 

A plataforma deve ser intuitiva, amigável para que o produtor e cliente possa ter acesso e clareza nas transações e compras, a plataforma deve ter um painel administrativo para que o produtor gerencie seus eventos de forma independente.
A logica de funcionamento deve ser pensada principalmente para mobile pois 87% dos acesso é por dispositivos moveis, porem não pode esquecer que do desktop.

A entrance surgiu em 2021 com intuito de ajudar os eventos de música eletrônica a ter um espaço dele, pois as grandes plataformas de venda de ingressos não dão a devida atenção para esse tipo de evento, começamos a operar em janeiro de 2024 onde realizamos 18 eventos durante o ano, porem a plataforma também funciona como agenda onde temos diversos eventos na agenda que não estão a venda com a plataforma, o que vem ajudando a criar uma comunidade de visitantes constantes, hoje já temos mais de 2000 usuários cadastrados e 70 produtores, aparecemos em cerca de 40.000 pesquisas no google mensais.

Requisitos Funcionais (O que o sistema faz)
Autenticação e Cadastro:
- O sistema deve permitir que o usuário faça login com e-mail e senha.
- O sistema deve permitir que o usuário se cadastre com informações como e-mail, senha, nome completo, cidade, estado, endereço, CEP, CPF e número de telefone.

Gerenciamento de Produtos e Eventos:
- O sistema deve permitir uma seção da aplicação como um gerenciador de produtos selecionados (carrinho).
- Após a compra do ingresso, o sistema deve gerar um QR code único válido como ingresso.
- O sistema permitirá que os usuários pesquisem sobre eventos cadastrados, utilizando filtros como estado, data, nome, vertentes ou tags.
- O sistema exibirá eventos passados apenas quando os usuários pesquisarem pelo nome do evento; caso contrário, mostrará apenas eventos futuros.

Exibição e Navegação:
- O sistema permitirá que os usuários vejam os próximos eventos à venda por meio de um carrossel.
- O sistema terá uma área de banners rotativos com as principais informações de eventos escolhidos pelo administrador.
- O sistema terá atalhos para a galeria de fotos.
- O sistema terá uma página completa de exibição de eventos cadastrados, incluindo um mapa digital com as localizações dos eventos.

Blog e Comunicação:
- O sistema terá uma área de blog com as últimas 9 matérias postadas.
- O sistema permitirá que os usuários se cadastrem, utilizando o e-mail, para receber newsletters.
- O sistema terá uma página "Sobre nós".

Galeria de Fotos:
- O sistema terá uma página de galeria de fotos, com álbuns separados por nome e data, mostrando os últimos álbuns criados primeiro.
- O sistema permitirá que os usuários pesquisem por álbuns usando filtros como nome, data e localização.


Requisitos Não Funcionais (Como o sistema é)
Segurança:
- O sistema deve implementar criptografia para senhas e dados sensíveis (por exemplo, CPF e endereço).
- Deve seguir práticas de segurança contra acessos não autorizados (ex.: proteção contra SQL Injection).

Usabilidade:
- A interface deve ser intuitiva, com design responsivo para diferentes dispositivos.
- A aplicação deve oferecer ajuda, como tutoriais ou FAQs.

Desempenho:
- O sistema deve carregar páginas e realizar operações principais em até 3 segundos.
- Deve suportar até 5.000 acessos simultâneos sem comprometer a performance.

Manutenibilidade e Escalabilidade:
- O código deve ser modular e seguir padrões para facilitar atualizações futuras.
- Deve permitir escalabilidade para acomodar novos eventos, fotos e usuários conforme o crescimento da aplicação.


Stakeholders e Tipos de Usuários
Usuário Visitante:
- Pode navegar na aplicação e fazer login/cadastro.

Usuário Cliente:
- Pode editar suas informações públicas e comprar produtos.
- Gerencia os produtos no carrinho.
- Visualiza fotos na galeria.
- Pode transferir ingressos para outros usuários clientes.

Usuário Produtor de Eventos:
- Realiza todas as ações do cliente.
- Cria e edita eventos, ingressos e define seus tipos (ex.: estacionamento, consumação, combo).
- Adquire plano de hospedagem de fotos e realiza uploads.

Usuário Suporte:
- Realiza ações do produtor de eventos.
- Altera dados de usuários, eventos, produtos e ingressos, incluindo inclusão, edição, transferência e desativação.

Usuário Administrador:
- Realiza todas as ações do suporte.
- Gera relatórios abrangentes sobre os dados do sistema.
- Altera níveis de acesso de todos os usuários.


Sugestões e Melhorias
- Subdivisão de Funcionalidades:- Divida "Cadastro" entre usuários gerais e produtores de eventos para facilitar testes e implementação.
- Separe "Gerenciador de Produtos" e "QR Code" em módulos independentes para manutenção.

- Requisitos Não Funcionais:- Adicione um requisito de acessibilidade para incluir pessoas com necessidades especiais na aplicação.
- Inclua critérios de compatibilidade com navegadores populares e versões.

- Stakeholders:- Considere criar níveis adicionais para clientes VIP ou convidados especiais, se isso for relevante para os eventos.
